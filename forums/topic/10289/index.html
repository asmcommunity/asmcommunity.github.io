Some small additions
I've added FAQ to the FASM website, if you have some suggestions about more Q&As that could be added there, please let me know.
I'm also attaching a small tool I've been using many times while tracing some bugs in the FASM, this is a tiny include for the console version of FASM (just put the "include 'fasmdbg.inc'" at the beginning of "fasm.asm" file), which displays name of each module when it's executed and stores the intermediate results of preprocessor and parser stages into separate files. It may help you getting some knowledge about how FASM works, before I finally write my promised "Guide to FASM internals".
