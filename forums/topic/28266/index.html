String Length Limitation
Did you define the whole string on one line? Like: 
longstring db "blah blah blah blah blah blah blah blah..and so on..",0
If yes just split it into multiple parts, the problem's just that masm can't handle very large definitions of strings, but strings can be as long as you want, they are just data with a 0-byte at the end. If you split the string into multiple parts: 

longstring  db "this is a "
            db "very long string",0
It will work. Thomas
