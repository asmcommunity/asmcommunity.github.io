There is a cheap fix for this problem
Since the dll enums the FMT cfg in the order they are in the actual
file you can put a null instruction before the ".code" line

.ie
"NMHDR.code",0,0 ; Ignore current line tab,ignore next lines
".code",*0,**     ; force current line tab size to 0,next lines reset to 'StartingSize'

The ".code" line would not be used in this case since A1Format will think it has already proccessed this line
This trick is also useful for the word Process getting mistaken for the word "Proc"
Im afraid 'NMHDR.code' slipped past me. :oops:
