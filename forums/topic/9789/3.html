RadASM has no method for using both. You can select one by modifying masm.ini section 




"-when you change some source there's no more that little square on the left uper part of edit-window"



Sounds like a serious bug. Is it easy to repeat? Do you have an examle?



"-the + or - sign (proc expand/collapse) covers bookmark square which by the way covers breakpoint

RadASM maintains it's own bookmarks and breakpoints so it is no problem to overlap +/- and or breakpoints. However only one of them will be painted. Since RAEdit uses an imagelist for painting bookmarks it will not help to make breakpoints smaller. I don't think this is a big problem tho.
