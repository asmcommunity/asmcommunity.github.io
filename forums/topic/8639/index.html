Printing and Print Dialogs
If you program is going to be a windows application, the main window is to be created and initially displayed by you in your program (look up win32 skeleton)
From this point, its up to you how much you want to let Windows draw your display, and how much you wish to "owner-draw" (more advanced)...
The alternative is to code your programs WITHOUT a window, which makes them "console applications" that put their output text in a 'DOS window'...
The choice is yours, but I suggest you begin with looking at examples.
