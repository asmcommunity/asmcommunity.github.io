Re: Remote desktop????
You can use CreateDIBSection, which makes a bitmap that will let you directly access the pixel values. DirectDraw stuff might give you better performance. And, ideally for a remote desktop app., you'd use a GDI Mirror Driver like Ultr@VNC does - but that's a pretty advanced topic.

VNC would be a good place to start, actually - it will give you an idea of the things you need to make this all work efficiently... not that VNC works efficiently as soon as you use it across the internet, nothing I've found beally beats Microsoft's RDP (remote desktop protocol), but that might as well be because of it's tight integration with the operating system.
