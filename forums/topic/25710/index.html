Re: allocate a dynamic number of bytes
First, forget Global/LocalAlloc (except for a few specific situations, like dealing with the clipboard), HeapAlloc is your friend - or VirtualAlloc for large allocations that you won't be resizing.

You can't allocate a "dymanic number of bytes in your exe file". What comes closest is, in MASM syntax, "myfluff BYTE 4069 dup (?)" - but that's a static build-time option, not a dynamic allocation.
