Unhandled exception errors are handled by the HLA run-time system that begins
the execution of an HLA program. It literally means you haven't put any try..endtry
blocks in your code to handle a given exception that gets raised.

The "excepts.hhf" header file is about as close as you'll get to anything that
deals with this exception; however, perusing that won't be very satisfactory to you,
I'm afraid.

The exception is most likely caused by an illegal memory access. Is there a hexadecimal
number displayed in the dialog box when the exception occurs? This is the exception
number.

The best way to track down problems like this is to run OllyDbg on the code. It does
a pretty good job of identifying the offending instruction.
