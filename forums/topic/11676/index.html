m32lib trim function bug
just thought i'd point out a bug in masm8, m32lib's trim.asm :

; --------------------------------------------------
; place terminator at position of last tab or space
; --------------------------------------------------
mov , byte ptr 0

this will overwrite the last character of the string with zero. here's the fix:

cmp , byte ptr 0
je @F
mov , byte ptr 0
@@:
