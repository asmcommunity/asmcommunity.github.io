Hidding Process
Some of the ways I know: (most of which are dirty)
9x: RegisterServiceProcess; hook ToolHelp APIs; global message hook
2000 and XP: global message hook; remote thread; hook NTDLL but lacks performance 'cause of copy-on-write.

Can also use WM_TIMER to run your code in another process (a quick way). Or modify IAT for GetMessage and PeekMessage to let the target thread run your code.
