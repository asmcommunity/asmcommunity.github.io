explorer`s weird way of handling filename length limitation
Just stumbled upon some weird 'feature' in Windows (XPPro SP2). Not sure if this something new, but I haven't noticed it earlier.

The limit of full filepath name is set to 247 chars. And when creating a new subdirectory in some folder, explorer will limit the subdirectory name length so it wont overflow (simply doesn't allow you to type any more characters). But looks like it doesn't checks length of parent directory name when renaming. So if parent directory renamed in a way that full path maximum length is exceeded subdirectory becomes inaccessible.
