Well, HLA v1.43/1.45 does support FASM output.
It is, however, in an experimental form so I won't make any guarantees
about the quality of the code emission other than I'd bet that something
*is* broken somewhere :-).

Seriously, though, as I was getting into some serious FASM testing,
I decided to change gears and rewrite the HLA internal code generation
code (to better support different assemblers like FASM and NASM).
Once I get this rewrite complete, finishing off the FASM support will be
one of my highest priorities.

My other high priority will be to update alink so that we've got access
to a free 32-bit linker. The combination of HLA, FASM, and ALINK, along
with a few other pieces of free software, will allow me to release a
"completely free" version of HLA for Windows (HLA is already completely
free under Linux, I'd like the same thing for Windows; btw, "free" means
free of copyright restrictions on code distribution; e.g., I can't legally
distribute MASM or MS-Link with HLA today).
Cheers,
