To franlou:

You can only have one .rc file in a project
directory.

If you have more then one you can rename the .rc(s)
to .z4 or something.

The program is not setup to have multiple .rc
files in a string, if the .rc compiler can
handle such a thing maybe someone could tell me
how to setup the command line.

You can have multiple .asm files, when you assemble
the project for the first time the Build program
will ask you which .asm files you would like to include.
After the first time it will keep track of the
files.

Normally in a project folder you have the .asm
file(s), the .rc and if needed the .inc and or .def
files that pertain to that project.

P.S. I used to have the Build program setup so
you could have multiple .rc files, but each .rc
file had to have the same name as the project
name that it pertained to, but someone talked me
out of that.
