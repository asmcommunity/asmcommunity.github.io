Multiline Macros (To Ernie, For Everyone)
xtreme, 1. Yes, if they are designed to write code, they will place a copy of the code at each place you call the macro. 2. NO, they are different things, macros can be used to pack unchanging code in convenient ways, inlining repeat code without all the typing etc .. Library modules are more efficient in size and convenience terms and can do more complex things so basically you select which does the job for you in the code you are writing, both have their place. 3. A library module can use a macro but macros cannot be compiled into a library of their own, a macro is basically a text expansion and are stored like source code. This is reliable and unproblematic.
