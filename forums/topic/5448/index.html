Help for GetOpenFileName
This member contains the zero based offset into the lpstrFile member of the same structure. If nFileExtension is 0 then either the file has no extention or the string ends with the dot (.) character.

Sooo
if nFileExtension is 0 skip processing...
else read next 3 bytes from and compair them to what you want.
