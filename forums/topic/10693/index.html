Upgrade for the INC2L.EXE utility
I have fixed a few problem with the last version and it is now versioned as Version 2.0.

I have added support for using an environment variable to specify the path for ML.EXE and LINK.EXE so that the utility can be used without assuming the MASM32 directory format.

If the environment variable is not set, it will look for ML.EXE and LINK.EXE in the \MASM32\BIN directory as before.

To get the syntax, just run the utility without any command line parameters.
