C++: Why all these freakin' data types?
The point sorely missed is:

Use what works right. Assembly sounds cool if you're working alone and have very few smartypants going through your code. Work in a team and you'll realize it really isn't so cool when others, expecially those who don't know it, have to read/debug your code. Write good and maintainable (and "portable") code as much as possible. No code can be *absolutely* portable, so an argument on that topic is utterly fruitless. A few quotes follow:

There are features that should not be used.
there are concepts that should not be exploited.
There are problems that should not be solved.
There are programs that should not be written.
?Richard Harter

Simplicity is the soul of efficiency.
?Austin Freeman

To keep large programs well structured, you either need superhuman will power, or proper language support for interfaces.
?Greg Nelson

Writing code ? is not an exercise in manliness.
?Mark Hahn

Programs must be written for people to read, and only incidentally for machines to execute.
