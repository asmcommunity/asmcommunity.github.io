Why are you trying to use interrupts with the Win32 API?
If you want to use DOS, then the API isn't available to you, and if you want to use Win32, then interrupts are a bad thing TM.

How do you use the GetCommandLine API?
Simple: 


.data?

pCmdLine   dd ?



.code

  invoke GetCommandLine

  mov pCmdLine, eax

Done!

Once you've done that, pCmdLine can be used like the address of any other string.

For example: 


  invoke GetCommandLine

  invoke MessageBox, NULL, eax, NULL, MB_OK

  invoke ExitProcess, 0
