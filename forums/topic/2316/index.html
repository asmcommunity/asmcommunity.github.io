Yes as long as you don't try and mix C++ compilers with assemblers of different brands. MASM will create library modules that can be used by Microsoft C/C++ and from memory, TASM can write modules for Borland C++ compilers.

You will need to ask one of the C++ guys for the syntax of prototypes for your particular C++ compiler. From memory it varies between C and C++ in Microsoft compilers.
