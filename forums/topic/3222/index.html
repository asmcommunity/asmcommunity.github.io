ExitProcess and QEditor
Quantum,

Normally you use ExitProcess to close and clean up the running application and you can expect a crash if you don't.

Qeditor uses the older API WinExec() instead of CreateProcess() to run external EXE files so this may be the reason why. I don't have any specific data on why you don't using it.

ExitProcess and QEditor
WinExec calls CreateProcess to do it's work, so it's nothing but an
inflexible interface to the real process creation stuff.

Quantum, if you don't do an ExitProcess, your app will keep on running
and executing <whatever>. You usually hit something that causes
a GPF pretty fast, if the app doesn't end in and endless loop ;).
That your app doesn't crash under QEditor is a weird thing, but not
something you should rely on :P. Just do the ExitProcess call.
