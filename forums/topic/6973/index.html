I build this reading example with plain socket communication, and it has the same error like your example:

If you communicate with HTTP1.1 servers, they have the possibility to tranfer the website
in chunks, called "Transfer-Mode: chunked". This involves some length info in front of the data,
and could have some time-gap between the chunks. So in this cases the read reads only the first
~ 2000 bytes of the page.
