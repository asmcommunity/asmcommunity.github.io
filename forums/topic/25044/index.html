Older version demo8.RES stops RADASM from updating RES file.
Hi, Biterider:

    To find the reason, I use two ways of re-installation. (1)Overwrite version 1.3e with version 1.4. (2)Complete un-install, delete the folder of OBJASM and reinstall. After hours of test, I found it's just an over-writing problem. When installing a new version that overwrites the old one, something unexpected happens. It is the Demo8.RES generated by older version makes such problem. RADASM V2.2.0.9 seems not to delete the old RES file if it exists. Therefore the older version problem happens again. When I delete the old Demo8.res, problems were resolved. I don't know why RADASM doen't delete the out-of-date demo8.RES automatically.

    The attached RES file below is the trouble making one, if you copy it to DEMO8 folder, then compile your source code again, you will probably see such problem.

    To sum up, supplied EXEs works fine as well as the source code. It's the old version DEMO8.RES stops RADASM from updating itself.

Regards,
