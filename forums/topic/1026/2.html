Hello NightShade !

It doesn't matter on modern CPU's if you test bytes or dwords from the point of time-use, but it's shorter in memory so the execution-block may be shorter ! (good ;) )

Using a lookup table is good if you want to change events often, but I wouldn't do that at the beginning of the windproc.
If you get a cache-miss it takes up to 60 cycles for the cpu to do the right fetch.

Another way to optimize code is to align all jump and calls to 16-byte-boundaries, because the cpu prefetches code in 16 byte-blocks. A jump or call forces the cpu to do a new prefetch, ..., well it is a bit like fetching dwords from odd addresses !
