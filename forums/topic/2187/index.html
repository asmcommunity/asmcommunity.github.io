LABEL keyword
The label keyword is to provide a broader range of possibilities when creating macro or pre-compiled scripting.

I posted an example earlier that built a large array from a pre-compiler for loop.

To do this easily, i made use of the LABEL directive to tell masm that before anything happens from here on, i want to "LABEL" this point in memory as 'some name', and can be thought of as a realPOINT type when referenced by code.

All this was done as: Some_label LABEL realPOINT

So masm will then take note that here lies a valid address label.

I then follow it up with a while loop that writes alot of realPoint data into the data sement (which is not given any label).

Once done, thet data sement would look like:

Some_Label:
realPoint .....
realPoint .....
...
realPoint .....

Hope this is clear enough... (kinda hard to explain :)
