Offline Browser Part 2
Well I updated the database and the offline browser so any 'sync but doesn't update' problems are out of this world now :) The offline database format has changed a bit as well. So you need to download the database again. If you already downloaded the previous install files then you can probably get away with downloading the executable only. The Database Just the executable file Partial Install (lacks MDAC and VB6 runtime/OLE automation) Full install The images The images need to be unzipped in a directory named 'Images' in the application directory. The database and executable file (if you choose to download this) needs to be unzipped in the application directory. Should you have downloaded the semi install and you notice that it still needs things, then you can download VB6 runtimes or Microsoft Data Access Components . You only need to install those once so if you have already installed those then don't bother with them again. If you haven't done this already, then you need to ad a DSN that points to the database: go to your control panel, look for ODBC32 and a User DSN named 'Offline Messageboard' and point it to the database. just click 'Add', Select Microsoft Access Driver and click finish. Set 'Offline Messageboard' as datasource name. Then click on select, locate the offline.mdb database in the install directory and select it. Then just click OK. That's it :) My sincere apologies to anyone needing to download the database again, it's a must do I'm afraid.
