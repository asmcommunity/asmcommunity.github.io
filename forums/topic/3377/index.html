OCX Container
Hi,

based on Xtremes webbrowser container I have made a more general container. To be more exact: I deleted all the webbrowser specific stuff, added interfaces some controls wanted to find and inserted a dialog to select one of the registered OCX controls. The result is something partly similiar to the "OCX test container" supplied with VC. But dont expect too much: its not a development tool to insert controls via D&D in a form and generate code with a mouse click (:) ). Its a simple ocx container (with source code included) written to support debugging of activex controls.
