About new "icon" macro...
Hi all.

The new "icon" macro solves the problem with using multipi icons in fasm application. But there is one little problem - the ICO file format allows to insert more than one image in ICO file. And usually ICO files contains image for different screen modes - for example 16 colors image and 256 colors image and also images with size 32x32 and 16x16.
So, I don't like to put different images in separate files and I wrote the macro "multiicon" that reads all images from ICO file and creates RT_ICONS and RT_GROUP_ICON resources simultaneous.

This is the first raw version and any improvements and notes are welcome. :)
