Guy,

I'm assuming you are trying to retrieve the text only portion of the email from the email message source. If you look at the source for this kind of email, it will include the following line:

Content-Type: multipart/alternative;

This tells your email reader that there are more than one "part" to the email and that it may have to process only one. The text part of the email is usually first followed by the HTML part which are separated by a "boundary"

You can find this by searching for the text:

boundary=

This will quite often be a very strange string. This boundary string is used to separate different "Content-Type" sections, for instance text/plain, text/html A good reason for having both plain and html sections is so that with one e-mail a user with a plain text e-mail program can view the text and a user with a graphical email program can see the html "page"

So to retrieve the text only portion, select the text after the line

Content-Type:text/plain

and a blank line and before the next "boundary"
