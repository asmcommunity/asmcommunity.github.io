Thanks for your kind words. From version 1.00.0.0030, to avoid compiling twice or more times the same resource, and to make it closer to the classic programming, the way to load an image file has changed. First, you have to add the resource in the image table. After that, all resource ID's will appear in the CursorIcon, IconBig, IconSmall and Picture properties so you can select one of them. For example, you define a resource named IDI_APP_ICON with a .ico file. Then, in the property you want to change, let's say IconBig, you assign the IDI_APP_ICON ID. Have a look at any of the examples.
