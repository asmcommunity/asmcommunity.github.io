Re: Iczelion's ODBC tutorial
ODBC access should be independed from the underlying database.

Make sure, that you have installed a valid ODBC driver for excel tables and check if you can access this ODBC source with some other tool, e.g. Microsoft "RowsetViewer" which is part of the free Microsoft Platform SDK.

With help of that external tool you can also check if all the commands you are using, are supported by the Excel ODBC driver.
