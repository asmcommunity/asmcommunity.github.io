To Randall Hyde: HLA & RadASM
Hi Randall,

just a couple of things...

First point:
In your documentation regarding the integration between RadASM and HLA there is a mistake. The section of RadASM.ini suggested by you is as follows: 



[F1-Help]

F1=$H\Win32.hlp

[COLOR=red]CF1=$H\PDF\HLARef.pdf

SF1=$H\pdf\HLAStdlib.pdf[/COLOR] 

CSF1=$H\Rc.hlp


but the settings for the PDF files are generating an error and a GPF of winhlp32.exe due to the fact that the F1 key calls WinHelpA. See the following thread for further info...

http://www.asmcommunity.net/board/index.php?topic=15022


Second point:
Browsing this board I've found a really interesting Iczelion Tutorials convertion done by Green Joe.

http://www.asmcommunity.net/board/index.php?topic=12998

These files are basically the same included in the HLA examples package, but they are better organized (I personally prefer a different folder per project instead of one messy folder containing everything) and ready to be used (except some small modifications to the .rap files) with RadASM. I think it would be interesting to integrate something similar in the next release of the examples package...

Anyway, what I've found in the thread are the tuts from 15 to 32. Was the same exercise done for the tuts from 2 to 14? If yes, are they available somewhere for download?

Green Joe: I agree that changing the MenuMake to "Build,,,Build All,Compile RC,Check Syntax,Run" in hla.ini can solve the issue of swapping the referred numbers in the rap files, but anyway you'll have to modify every single rap files, as the line "Menu=1,1,1,1,1,0,0,0" should become "Menu=1,0,0,1,1,1,1,0", otherwise the "Check Syntax" & "Run" options are not enabled.
