Hi Ketil,

The problem with building modules has been fixed in GoAsm 0.49.10 beta, the /C argument will place the OBJ files in the current directory. However I have noticed this in the LINK command...

9,OT,C:\GoAsm\lib\LIB.EXE /OUT:"$9",*.obj

Gives...

C:\GoAsm\lib\LIB.EXE /OUT:"Convert.lib" "3.obj"

To correct it I have to add the following

9,OT,C:\GoAsm\lib\LIB.EXE /OUT:"$9",.\*.obj
