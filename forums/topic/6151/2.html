RadASM 1.2.0 (06-19-2002) Bugtest
Hi all

Fixed a GPF bug in dialog editor. RadASM would GPF if a control was off top of client area. Also fixed other minor bugs.

Added new features:

- New addin message AIM_SETSTYLE:
. Parameters: hWin: Handle of property listbox
. wParam: Pointer to DIALOG struct.
. lParam: TRUE if ExStyles. Return TRUE and RadASM will not show style
. change window.
- Added dialog editor option: Simple Property
- Added code editor option: Always on top
