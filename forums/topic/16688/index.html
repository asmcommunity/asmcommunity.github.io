mov pa.hwnd, hWnd
You can't do two indirect accesses with the x86 instruction set - there's no "mov ," addressing mode. Thus, you'll have to either mov the hwnd to a register first, or do (slower + two bytes longer) push/pop combination.

Unlike better architectures like the Motorola 68000 series, Intel CPU's are not able to do direct memory to memory transfers, except for the PUSH/POP and MOVS instructions. Intel is also woefully short on general registers too. Ratch
