As well, i just had a thought that you might want to explore in the next version.

The error reports are nice, but every so often even these dont help cause they are witten very vagely to be applicable to all code. My *idea* was to extend the tool to have a 'appended notes' to the desciption. Perhaps set up a directory/reg key to log user defined notes. You could even use the zlib compression tool to compress these notes till needed.

So if (for example) i had a frustrating error, and the general error desc is still too vague. Then i eventually figure it out after 2 hours. I can go back and append a note to myself saying "It was this cause of that, specifically".

Then 20 hang-over's later, when i get this error again, i look up this error code and see there is a note attached (perhaps an enabled icon/button), i read the note as well, and discover I did it again, and i got the error fixed in 20 seconds.

Its a bit more work, i realize, but it would be both a good coding project and practice. As well as a practical feature.
