open file from shell bug
Using 2.0.4.3 bugfix E, there seems to be a bug when opening files associated with RadASM through the windows shell.

When Windows creates the association, it puts quotes around the command string. Something like:

"C:\RadASM\RadASM.exe" "%1"

Taking all of the quotes out of the string fixes the problem, but I'm not exactly sure why it wouldn't work in the first place.

Now that I think about it, I'm pretty sure this only started happening after I installed SP2.
