How to make a custom control
You don't need to use any dialog editor.
Simply,build a simple standard .exe,add a new .asm file and in this new .asm file
make your control.In this file register a new class and create a child window which
will has as parent the main window of .exe.So,you can test your control by runing
the .exe.
After finish with control creation,build a DLL and transfer the code of new .asm file
in the DLL.

Custom controls are nothing more than a container process to wrap up the "behavior" of a visual control. There is nothing stopping your from making alot of child Edit controls for a custom control. You can even make dialog boxes if you must, but dont forget that a control is ment to be the child of another window. This means in 99% of all cases it would NOT be a dialog, but rather a custom drawing window, with background painting, and child edit boxes and anything else you might need to encapsulate its "behavior".
