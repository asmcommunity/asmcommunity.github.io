2- I have a suggestion about unicode char:
- For easier job, RADASM only support reading unicode char under Win2k and WinXP, not support Win9x, ME.
- You only need write a detector of unicode char, if found, don't try to modify or save file because if you do it, everything in unicode will be crashed. ( I tried and it happened )
- When you found unicode char in source file, use notepad.exe to open it ( because in Win2k and WinXP, notepad based in unicode) . You can also write a plugin to read and write unicode source. It's not so hard and not complicate too, because when programming I only saw unicode source appear in rsrc.rc. With rsrc.rc we only need to modify just for a little, not need to highlight or auto complete like normal source code.
