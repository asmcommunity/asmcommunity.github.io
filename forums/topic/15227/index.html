AddProc bug fix plus new ini name
Hi all,
EDIT:
AddProc now saves its settings to an initialisation file called WAAddins.ini in its own section called AddProc. It appears this is going to be the ini file format for Winasm addins (see http://www.asmcommunity.net/board/index.php?topic=15223 as well as Akyprians post below).

There was a bug that caused a GPF when adding a new procedure to a .inc file or a new (empty) file which has now been fixed.
