Indentifying Kernel32.dll I/O APIs
Every function exported by Kernel32 is in the Kernel32.dll file so you can get the text versions of the functions from that with a hex editor and a bit of ingenuity. I originally started to learn the API by dumping the function lists from the bigger api files. If you want to know which functions of the Kernel are exported you can check the PE, it will contain a list of each api's function called and the dll it's called from as well.
