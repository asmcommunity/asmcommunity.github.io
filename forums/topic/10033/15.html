Hi cakmak

The assembler.ini files have been changed in RadASM 2.0.2.8. If you manually change the ini file then do the following:

Section

MenuMake=Compile RC,Assemble,Link,Build,Go,Run,Run w/debug,Go All,Assemble Modules

In section add:

MenuMake=1,1,1,1,1,1,1,1,1,0
7=0,0,\OllyDbg\OllyDbg,$.exe
17=0,0,\OllyDbg\OllyDbg,$.exe

For each of the project types add:

MenuMake=1,1,1,1,1,1,1,1,1,0 ;Change to what fits the project type
7=0,0,\OllyDbg\OllyDbg,5
17=0,0,\OllyDbg\OllyDbg,5

Chane \OllyDbg\OllyDbg if you use another debugger.

In section change:
1=&Compile RC,55,M,1
2=&Assemble,31,M,2
3=&Link,91,M,3
4=&Build,103,M,2,3
5=-,0,M,
6=&Go,43,M,2,3,4
7=-,0,M,
8=&Run,67,M,4
9=Run w/&Debug,4,M,7
10=-,0,M,
11=Go &All,79,M,1,2,3,4
12=-,0,M,
13=Assemble &Modules,0,M,6

Also there was a bug in the first upload of RadASM 2.0.2.8 that caused Assemble Modules to not be active.

Hi Ketil,

RadASM will GPF if you select select collapsed text in an edit window to the top of the window and delete or overwrite the text.

To reproduce the bug:
    [*]Open a project with collapsed procs
    [*]Press ctrl-A or Select a collapsed proc and everything to the top of the edit
    [*]Press Backspace or Delete or type a character
    This is reproducible in Win2K and 98SE. I removed the following AddIns to make sure that it wasn't them and it still GPF'ed:
      [*]Colref.dll
      [*]Version-Inc.dll
      [*]style-manager.dll
      [*]RES2DLG.dll
      [*]NoteTools.dll
      [*]AsmVars.dll
      [*]MakeDef.dll
      [*]Bitmask.dll
      [*]ButtonChanger.dll
      This bug does not show up if a collapsed proc is not in the selected text.
