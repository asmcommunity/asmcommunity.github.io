Hello,
I don't create dialog boxes and procs,
because I don't have the knowledge.
Perhaps you can use CreateWindowEx on
WM_INITDIALOG inside the DIALOGPROC.

It is my prefered programing style to use
the resource only if I don't find another way.

It is not a religion, it simply my style ...

In my skinner example I use tables for
creating multiple windows because my asm code
becomes compacter and I can use a pointer
to access values like the window handle.
For example on WM_CREATE you can use a loop
to create multiple STATIC windows.
It is much more compacter than creating xx
STATIC windows one after another.

But, there is always a way to make it better.

Here are some more predefined window classes:

ToolTipsClassName ------> Tooltips_class32
SysListView32ClassName -> SysListView32
ProgressClassName ------> msctls_progress32
StatusClassName --------> msctls_statusbar32
