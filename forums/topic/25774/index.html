Re: RadAsm & XP Manifest
Well, a res file has more than just what you see. Radasm does probably a good job kicking out the non displaying characters. If you modify something and then save Radasm probably doesn't save the "invalid" characters.

Either you use a hex editor to modify res files - and watch out with that as a res file has a _very_ strict format with forward and backward references all over the damn place... Or you modify the rc file directly which is what I would recommend.

Re: RadAsm & XP Manifest
Radasm doesn't make any modifications to the res file. The res file is a compiled rc file. If you add an xml in Radasm then Radasm adds it to the rc file and when you compile it - it compiles the rc file into a res file. The res file then get's added to the executable by the linker.
