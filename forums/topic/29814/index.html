Re: Question about WndProc
If you noticed, WndProc(or whatever you named it) is mapped to a class and window is created out of this class. So, as long as you create more windows using one registered class, all the messages will be processed by the same WndProc that is mapped to the class.

If you want messages to a window processed by a different WndProc, create another class, register that class and create window out of that class

Check out the tutorial at http://www.winprog.org/tutorial/ for an example.
