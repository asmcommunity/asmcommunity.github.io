All memory locations store in binary.  Essently everything is binary.  A 32 bit register is just that, 32 binary bits grouped together.


I assume the Irvine32.inc provides the DOS keyboard commands for getting user input.  This will enter as a ASCII string of text, pointed by an address pointer.  To convert the Hex string to Binary, you will have to parse the memory pointer, one byte at a time and convert each byte to its binary couterpart, eight bytes of either 30h or 31h.

Alternately, you can use the SHL and test for Cary bits on each byte from the entered Hex string.  In this case, you wont need to provide 16 - 8 byte output patterns of ASCII 1's/0's.
