WinDbg problem
If you are using Microsoft's Windbg, then these are the
command line options that are working for me. 
ML.EXE /c /coff /Zi /Zd project.asm

LINK.EXE /SUBSYSTEM:WINDOWS /DEBUG /DEBUGTYPE:CV /PDB:project.pdb /PDBTYPE:SEPT project.obj project.res

Replace "project" with your own file names.
