Splash window
Splash window

A splash window is usually used when an application is started and showed some information and or logo. The real use of the splash window is to distract the user’s attention while the application is doing its initialization work. A classical example of a splash window can be seen when you open a pdf file.

If you design a splash window you have pay attention to some rules. To capture the attention of the user you can apply some tricks. One of them is, if you know the preferences of your potential users, to select a motif that certainly will be looked at. Another trick is to give the splash window a different look and feel from the OS or some special effect.
How long should the splash be shown? It is recommended that it doesn’t remain longer than 4 seconds. To accomplish a good performance, the initialization of the main application should be started as soon as possible to gain some time. This can be done using a thread to display the splash window while the main application is doing its job.

The example I attached to this post shows an implementation using an ObjAsm32 class (Splash). Since the majority of the members here are men, I selected a motif that will catch their attention. As an example, if we take the Adobe Reader, the motif they selected is designed to identify the customers with the product. This is important for a commercial product and has to be considered during the design phase.
