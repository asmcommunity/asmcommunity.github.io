Afternoon, Vaxon.

No such thing as DX8.1 DirectDraw:alright: .

DirectDraw and D3d were merged in DX8 to become DirectGraphics.
You'll also notice that the ddraw examples in the ms or dx sdk actually use dx7 code.

Bit of a pain including dx7 examples with no dx7 directdraw sdk section in the dx8 sdk though:rolleyes: .

If you want to use DirectDraw-specific code, you'll have to use DX7.

The Platform SDK includes the latest DX version SDK at that time (my PSDK is Feb2k1 which had DX8a).
You can also download just the DX SDK from Microsoft (i.e. DX8.1 SDK).
