RadASM Help does not start, if a project is open
Hy,


I downloaded the RadASM 2.4.0.2 and I would like to report the following problem:

If the IDE is empty (there isn't open any project), than clicking on the Help->RadASM menu item will start the help. BUT if I open a project, clicking on the same menu item will not start the help file, in fact it does nothing. If I close the project the help file can be started again.

Is this a bug, or there is something that I don't know?
Please let me know, if there is a solution for this problem.

RadASM Help does not start, if a project is open
Hi bszente,

The help menu is specific to each assembler so you may have a different entry for the assembler you are using. Try this :

1. Start RadASM no project
2. click on Option > Help Menu
3. Check the entry for RadASM
4. open a project and follow the same steps
5. change the entry for the assember to the same as the one for no project

Thanks Donkey!


I found the problem: by default (after installatio) the RadASM help file was set to the "\RadASM\Help\RadASM.chm" path.
Since my project was on the D: drive, the file couldn't be found. But when the project was closed, the path referred to the C: drive letter and the file could be found.

A changed the path to "$R\Help\RadASM.chm" and the problem was solved.

I hope that the future RadASM packages will contain the correct path.
