The source code for the FpuAtoFL procedure in the FPULIB is also included in the MASM32v8 package. That procedure also allows the string to be in scientific notation.

The only detail to consider will be to use the procedure with the direction of leaving the converted float on the FPU and then store it yourself in whatever float format you require (i.e. REAL4 per your request).
