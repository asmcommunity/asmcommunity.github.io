Re: tasm ide
In my opinion RADASM is very difficult to configure, especially concerning compiler-related menu entries.
In general I avoid these annoyances by not using the menu entries for creating the final EXE but create a .BAT-file which contains all required commands which I invoke by a console window separately opened in the background.
RADASM does have some difficulties in recognizing some structures, comments and other <start>...<end>-like structures. Nevertheless I like it very much because of its syntax hilighting and project management features.
For full compatibility I simply copied all MASM*.INI-files to TASM*.INI.
TASM supports most of all MASM-features.
